## Design Context

### Users
Primary users are builders prototyping ChatKit-powered workflows, plus domain operators (e.g., legal analysts in this customized variant) who need to validate a reasoning workflow quickly in browser. They typically run short, iterative sessions and compare structured outputs (chat + visualization) before sharing or deploying.

### Brand Personality
Precise, confident, and editorial. The interface should feel trustworthy and instrument-like rather than playful, while still feeling crafted and modern.

### Aesthetic Direction
A warm technical editorial look: tinted neutrals, restrained accent colors, serif-forward headings, and layered panels that signal “analysis workspace.” Avoid generic SaaS cards, neon dark themes, and gimmicky effects.

### Design Principles
1. Preserve cognitive hierarchy: one dominant heading, lightweight context labels, and intentional contrast between analysis surfaces.
2. Build a cohesive token system first (color, borders, radius, motion), then apply consistently across components.
3. Use purposeful motion only for orientation (entry rhythm, subtle ambient movement) and honor reduced-motion preferences.
4. Prioritize scanability for expert users: compact metadata chips, explicit workflow sequencing, and clear panel framing.
5. Keep decorative elements restrained and semantic; every effect must reinforce structure or emphasis.
